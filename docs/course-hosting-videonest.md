# Course Hosting & VideoNest (WO-2026-004)

## Implemented

- Course CRUD API:
  - `GET|POST /api/courses`
  - `GET|PATCH|DELETE /api/courses/:courseId` (`DELETE` archives)
- Nested Module/Lesson CRUD API:
  - `GET|POST /api/courses/:courseId/modules`
  - `GET|PATCH|DELETE /api/courses/:courseId/modules/:moduleId`
  - `GET|POST /api/courses/:courseId/modules/:moduleId/lessons`
  - `GET|PATCH|DELETE /api/courses/:courseId/modules/:moduleId/lessons/:lessonId`
- Enrollment flow:
  - `POST /api/courses/:courseId/enroll`
- Video progress tracking:
  - `GET|POST /api/courses/:courseId/lessons/:lessonId/progress`
  - Tracks `percent_complete`, `last_position_seconds`, `watch_time_seconds`, and completion source (`manual`, `time-based`, `quiz-pass`)
- Drip scheduling:
  - Supports per-lesson `metadata.release_days_after_enrollment` / `drip_days`
  - Supports sequential unlock mode via course metadata (`drip_mode: "sequential"` default)
- Student dashboard UI:
  - `/dashboard` shows enrolled courses, progress bars, resume links, and available course enrollment
  - API payload available at `GET /api/student/dashboard`
- Lesson UI:
  - `/courses/:courseId/lessons/:lessonId` shows player + progress controls + lock state
- Certificate generation + verification:
  - `GET /api/courses/:courseId/certificate` returns an auto-generated PDF
  - `GET /api/courses/:courseId/certificate?format=json` returns certificate metadata
  - `GET /api/certificates/verify/:code` verifies certificate code
- Instructor admin panel:
  - `/instructor/courses` for course/module/lesson management
- Multi-brand visibility:
  - `courses_brands` is used as the visibility gate for student-facing course lists/access

## VideoNest Note

VideoNest API details are not present in this repository. A placeholder-compatible embed is implemented via lesson metadata:

- `metadata.videonest_embed_url`
- `metadata.videonest_asset_id` (mapped to placeholder URL format)

This follows WO stop conditions for unavailable/undocumented VideoNest APIs.

## Certificate Note

PDF certificates are generated by a lightweight in-app PDF builder (no extra dependency), each with a unique verification code (`CERT-...`).
